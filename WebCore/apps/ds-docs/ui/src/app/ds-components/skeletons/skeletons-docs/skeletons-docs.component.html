# Loading Skeletons

Loading skeletons act as a placeholder while a widget loads. They create the effect of reduced load times as well as reassure the user that content is loading.

## Use When creating a widget with either the ds-dashboard-widget (seen on the Analytics Dashboard) or the ds-widget, (seen on the Nps Dashboard) use skeletons as a loading animation while the API returns the data.

# How To Add a Loading Skeleton - Add a boolean to your ts file that is initialized to false. Set it to true after the api has returned the data for the widget. - Add [loaded]="myBoolean" to the 'ds-widget' level of the markup (where myBoolean is from the previous step) - Add skeleton="example-skeleton" to the 'ds-widget' level of the markup. Names of skeletons are below. Values | | Data Type | | ----------| --------- | --------- | ------------------------------------------------------------------------------- | `loaded` | Required | Boolean | Whether or not your content is loaded. Skeleton will show while loaded = false | `skeleton`| Required | string | The name of the skeleton | # Button Card Skeleton: skeleton="button-card-skeleton"
# Siple Card Skeleton: skeleton="simple-card-skeleton"
