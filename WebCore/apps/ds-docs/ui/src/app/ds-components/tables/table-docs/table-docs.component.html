# Tables #### Pagination Use if the returned data will be larger than 10 records. #### Sorting All tables should sort #### Column Alignment - Strings are left aligned - Numbers, including currency, are right aligned - Dates are right aligned - Icons are centered - Checkboxes are centered ## Formatting Directives Directive | | Description | ------------ | --------------------- |------------------------------------------------------------------------------------------------------------------------------- | `formGroup` | `ds-table-container` | Changes styles if applied based on form status. Example: If no `formGroup` is passed, the table will have view styles applied. | ## Formatting Directives Directive | | Description | ----------------- | --------------------------- |--------------------------------------------------------------------------------------------------- | `alignRight` | `ds-header-cell`, `ds-cell` | Aligns content to the right | `alignCenter` | `ds-header-cell`, `ds-cell` | Aligns content to the center | `AlignViewRight` | `ds-header-cell`, `ds-cell` | Aligns content only when in view state, ex. right aligned data like currency | `tableCheckbox` | `ds-header-cell`, `ds-cell` | Sets width and alignment for table checkboxes | `TableAction` | `ds-header-cell`, `ds-cell` | Sets width and alignment for action coulmns | ## View View state has a yellow row highlight and bottom borders.
## View and Edit Edit state removes the row highlight and adds side borders. Use `formGroup` on `dsTableContainer` to apply view and edit styles without additional CSS. `.view` is applied if there is no `formGroup` or if the form is present but disabled. Adding rows will autoFocus the first input.
## Validation The onLoad directive shows the tooltip when errors are shown. Validation should update on change.
